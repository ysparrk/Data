HyperText(Link) Markup Language

- xml: 확장가능한 마크업 문서의 장점(텍스트 간의 구분, 종속관계 => 그래픽으로 표현하기 좋다(DOM Tree))을 이용, link를 이용해서 다른 서버 또는 같은 서버 내의 다른 리소스를 참조가능(HyperText) 텍스트를 전달하기 위해 만들어졌기 때문에 text는 말단 노드이다.

- 브라우저: 정보전달을 한다 -> 그래픽을 그린다 -> 정보가 구체적으로 표현되기 위해 구조가 필요하다. - html은 그래픽으로 표현하기 좋다. - 크게 블럭O/블럭X 인지 구분한다.

CSS Cascading Style Sheets

- cascading: 계단식으로 위에서 아래로 쏟아지는 -> 스타일 적용에 우선순위가 존재한다 - html을 꾸며주는 양식 또는 문서

CSS Selector(선택자) 요소 간의 관계를 표현하는 것

  • div = div 요소 선택
  • div.red = class가 red인 요소 선택
  • #query = id가 query인 요소 선택(id는 보통 html 문서에 1개만 존재)
  • 자손 -> form input = form 요소 안에 있는 모든 자손(자식의 자식의 ...)인 input
  • 자식 -> form>input = form 요소의 자식 요소인 input
  • input[type='text'] = input인데 속성이 type이 text인것
  • :nth-child(n) : 몇번째 요소
  • div, li or .red, .blue, .color = 여러 요소를 동시에 선택
  • div+p = div 요소에 인접한 형제 p 요소
  • div~p = div 요소의 형제들 중에 p 요소

URL uniform resource locator 리소스의 주소

- 리소스: 인터넷에서 찾을 수 있는 자원 - https://comic.naver.com/webtoon/detail?titleId=827319&no=1&week=mon - https://search.naver.com/search.naver?where=nexearch&sm=top_hty&fbm=0&ie=utf8&query=%EC%95%88%EB%85%95 = 안녕 /url 인코딩(한글이라서)

  • https : http 통신으로 리소스를 요청하겠다.
  • comic.naver.com : (도메인) 서버의 주소 IP를 도메인으로 가림
  • webtoon/detail : path 리소스의 위치
  • ?titleId=827319&no=1&week=mon : parameter / 페이지를 동적으로 만들어준다
  • ? : parameter 시작, & : parameter 구분자, key=value
